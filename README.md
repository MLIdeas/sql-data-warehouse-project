# Data Warehouse and Analytics Project

Welcome to the **Data Warehouse and Analytics Project** repository
Part 1 — Data Engineering (Data Warehouse)
Objective

Build a modern SQL Server data warehouse that consolidates sales data from multiple source systems and provides a reliable foundation for analytics.

Part 1 — Data Engineering (Data Warehouse)
Objective

Build a modern SQL Server data warehouse that consolidates sales data from multiple source systems and provides a reliable foundation for analytics.

**Requirements**
**Data Sources**
ERP system (CSV files)
CRM system (CSV files)

**Data Quality**
Clean and standardize data
Resolve missing values
Remove duplicates
Fix inconsistent formats

**Data Integration**
Merge ERP and CRM data
Create a single business-friendly analytical model

**Scope**
Latest data only
No historical tracking required
No Slowly Changing Dimensions (SCD)

**Documentation**
Document architecture
Document data model
Document transformation rules


Part 2 — Data Analytics & Reporting
Objective

Develop SQL-based analytics that generate business insights.

Areas of Analysis
Customer Behavior

Examples:

Top customers by revenue
Customer segmentation
Repeat customers
Customer lifetime value
Geographic customer analysis
Product Performance

Examples:

Best-selling products
Revenue by product category
Low-performing products
Product profitability
Sales Trends

Examples:

Monthly sales trends
Quarterly performance
Seasonal patterns
Growth rates
Revenue trends over time
Expected Deliverables
Data Warehouse

Typically consists of:

Bronze Layer

Raw ERP and CRM data

Silver Layer

Cleaned and transformed data

Gold Layer

Business-ready dimensional model

Example:

Dimensions

dim_customer
dim_product
dim_location

Facts

fact_sales
Analytics Layer

SQL queries producing:

**Customer KPIs**
Total customers
Active customers
Average order value

**Product KPIs**
Top 10 products
Revenue by category

**Sales KPIs**
Total revenue
Monthly growth %
Year-over-year growth
